GitHub Pages Publishing

⚠️ Developer-only: these credentials get baked directly into this Admin HTML file. Anyone who receives this file can publish to the configured GitHub repo. Only ship to trusted Admin users.

💡 Tip: For each client, create a free GitHub Organization named after the client (e.g. phm-quotes, prk-quotes). The org name becomes the URL — https://phm-quotes.github.io/… — so your personal GitHub username never appears. You stay the owner of the org behind the scenes.

▸ Per-client setup walkthrough (do this once per client)
  1. Sign in to github.com with your personal account.
  2. Create an Organization: Click + (top right) → New organization → choose the Free plan. Org name = your client's company name in URL-friendly form (e.g. phm-quotes). You can name it anything — the org name will be the start of the public URL.
  3. Create the repo: Inside the new org, click + → New repository. Name it something like calculator. Set Public. Check "Add a README". Create.
  4. Enable Pages: In the new repo, click Settings → Pages. Source = Deploy from a branch, Branch = main, Folder = / (root). Save. (URL will be https://[org-name].github.io/[repo]/.)
  5. Generate a token: Click your profile icon → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token. Set Resource owner = the new org. Repository access → Only select repositories = your new repo. Under Repository permissions → Contents, set Read and write. Generate. Copy the token.
  6. Bake the file: Fill in the fields above (Owner = org name, Repo = repo name, Token = the one you just copied). Click 📥 Bake into ADMIN file. Save the downloaded file as e.g. PHM Quote ADMIN (for PHM).html. Ship that file to your client.
  7. Done. Your client opens that file, clicks 🌐 Publish to Web, and gets the URL https://[org-name].github.io/[repo]/ to email to their people. They never see GitHub, you, or any tokens.

For the next client, repeat with a new org name. One personal GitHub account = unlimited orgs = unlimited clients, each with their own URL.
